## 1. 배경

### 1.1 v0.0.1 의 scope (founding standalone)

본 repo 는 **founding commit (M-v0.0.1-alpha)** 부터 standalone 으로 운영:

- frontend shell (SvelteKit 2 + Svelte 5 + vitest 21 src file) standalone 운영
- backend spec (FastAPI + OKF + Pi LLM enrich placeholder) standalone 구현
- 5종 source plugin (mock + Gitea 4) — vendor-neutral, 외부 의존 옵션
- Path Y caller-provided user context (optional) — OIDC 없음
- storage: file-based default (M-v0.0.4+ 부터 DB 듀얼 옵션)
- API versioning: `/api/v1/` 시작 (M-v0.0.6+ 부터 v2 dual endpoint)

### 1.2 standalone 정책

본 repo 는 **완전 standalone** 운영 (ADR-0001 §2.3):

- 다른 backend / microservice 연결 ❌
- OIDC / SSO / Keycloak 등 인증 게이트웨이 ❌
- 사내 인프라 / private network 직접 의존 ❌
- vendor lock-in (특정 cloud provider 종속) ❌
- Network: 3 단계 (dev 사외 / staging 사내 / production 사내)
- Storage: file-based (M-v0.0.2+ default) + DB-based 듀얼 모드 (M-v0.0.4+ 옵션)
- Tier: **사외** (사내 한정 정보 0)

본 roadmap 의 결정 = 본 repo 의 ADR-0001 (founding charter) + ADR-0002 (OKF v0.1 채택):
